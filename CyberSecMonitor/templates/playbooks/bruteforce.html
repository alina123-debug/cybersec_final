Playbook: Brute Force

Investigation guide: check false positive first, then collect evidence, then contain/eradicate, then lessons learned.
1) False Positive check
  • Is source IP internal/VPN/known admin jumpbox?
  • Is the username a service account with expected rotations?
  • Are there maintenance windows?
2) Evidence to collect
  • source_ip, username, attempts, time window
  • successful login events after failed attempts
  • geo/ASN reputation (optional)
3) Example SIEM queries (5)
auth.failed where src_ip == <IP> and user == <USER> last 1h
auth.success where user == <USER> and after == incident_time - 10m
windows.security_event where event_id in (4625,4624) and ip == <IP>
firewall.deny where src_ip == <IP> last 24h
users.lockouts where user == <USER> last 7d
4) Response
  1. Confirm brute-force pattern (rate + failures).
  2. Block IP (temporary) if malicious.
  3. Force password reset / MFA check for targeted account.
  4. Create SIEM case + notify client owners.
5) SLA

Critical: 15–60m • High: 1–4h • Medium: 4–8h • Low: 24–72h
